## Por que este repositório existe

Muita gente pensa que o Code.org é "só blocos". Mas o **Game Lab** e o **App Lab** são JavaScript de verdade — o mesmo alternador blocos ↔ texto que uso com minhas turmas para fazer a travessia do concreto ao abstrato. Estes projetos foram criados **em aula e para aulas**: os comentários no código (alguns marcam até "aula 1", "aula 2", "aula 3") estão preservados exatamente como foram usados, porque código de sala de aula é documento pedagógico, não vitrine polida.

E tem uma camada a mais, que é a minha praia: cada projeto rende uma conversa de **educação midiática** — sobre as decisões de design, as economias dos jogos e o que as telas fazem com a gente.

## Como remixar

1. Abra o link do projeto no Code.org
2. Clique em **Remix** (é preciso estar logado — contas de estudante funcionam)
3. O projeto vira **seu**: mude sprites, valores, regras — e compare com o código aqui do repositório para entender o que cada mudança faz
4. No editor, alterne entre **blocos e texto** no botão do canto: é o mesmo programa nas duas formas

## Para educadores

Cada pasta tem um `README.md` com: o que o projeto ensina, as decisões de design que valem conversa em sala e uma pergunta de **leitura crítica de mídia** para o fechamento. O arquivo `codigo.js` é o código exatamente como está no Code.org — imperfeições incluídas, porque é assim que código de aprendizagem se parece.

A trilha completa que uso (do concreto ao abstrato): **Code.org → [GDevelop](https://github.com/paulapeclat/primeiro-jogo-gdevelop) → [Roblox Studio](https://github.com/paulapeclat/primeiro-jogo-roblox)**.
