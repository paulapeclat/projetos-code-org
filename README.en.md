## Why this repository exists

Many people think Code.org is "just blocks". But **Game Lab** and **App Lab** are real JavaScript — the same blocks ↔ text toggle I use with my groups to cross from concrete to abstract. These projects were created **in class and for classes**: the comments in the code (some even mark "aula 1", "aula 2", "aula 3" — "lesson 1/2/3" in Portuguese) are preserved exactly as they were used, because classroom code is a pedagogical document, not a polished showcase.

And there is one more layer, which is my home turf: each project feeds a **media literacy** conversation — about design decisions, game economies, and what screens do to us.

## How to remix

1. Open the project link on Code.org
2. Click **Remix** (you need to be logged in — student accounts work)
3. The project becomes **yours**: change sprites, values, rules — and compare with the code in this repository to understand what each change does
4. In the editor, toggle between **blocks and text** with the corner button: it is the same program in both forms

## For educators

Each folder has a `README.md` (in Portuguese) with: what the project teaches, the design decisions worth discussing in class, and a **critical media reading** question for the closing. The `codigo.js` file is the code exactly as it is on Code.org — imperfections included, because that is what learning code looks like.

The full track I use (concrete to abstract): **Code.org → [GDevelop](https://github.com/paulapeclat/primeiro-jogo-gdevelop) → [Roblox Studio](https://github.com/paulapeclat/primeiro-jogo-roblox)**.
